Операционные системы можно классифицировать по различным критериям:

По типу пользовательского интерфейса:

  • Графические (GUI): предоставляют визуальные элементы управления, такие как окна, иконки и меню. Примеры: Windows, macOS.
  • Текстовые (CLI): позволяют пользователю взаимодействовать с системой через текстовые команды. Пример: Linux (через терминал).

По способу обработки данных:

  • Однозадачные: могут выполнять только одну задачу в любой момент времени. Например, DOS (Disk Operating System) — была одной из первых операционных систем, использовавшихся на персональных компьютерах, и могла обрабатывать только одну задачу за раз.
  • Многозадачные: могут выполнять несколько задач одновременно (например, Windows, Linux).

По архитектуре:

  • Однопользовательские: предназначены для одного пользователя (например, Windows для ПК).
  • Многопользовательские: поддерживают работу нескольких пользователей одновременно (например, Unix, система, разработанная для обеспечения многопользовательского доступа и высокой надежности, ставшая основой для многих современных ОС).

По назначению:

  • Общие: универсальные ОС для настольных ПК и серверов (например, Windows, Linux).
  • Встраиваемые: используются в специфических устройствах (например, RTOS).
  • Серверные: оптимизированные для серверов (например, Windows Server, Red Hat Enterprise Linux).
  • RTOS: (Real-Time Operating System) — операционные системы реального времени, используемые в системах, требующих гарантированной реакции в строго определённых сроках (например, в медицинском оборудовании или автомобильной электронике).

По уровню абстракции:

  • Низкоуровневые: ближе к аппаратному обеспечению (например, FreeRTOS, который предназначен для встраиваемых систем и обеспечивает минимальные задержки в выполнении задач).
  • Высокоуровневые: абстрагируют аппаратные детали (например, Windows, macOS).

По поддерживаемым платформам:

  • Кроссплатформенные: работают на различных аппаратных платформах (например, Java OS, предназначенная для запуска приложений на разных устройствах, использующих виртуальную машину Java).
  • Специфические для платформы: предназначены для конкретного оборудования (например, iOS для устройств Apple).
